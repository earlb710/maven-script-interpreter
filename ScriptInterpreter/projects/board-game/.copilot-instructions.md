# AI Instructions for board game

## Project Overview

This is an EBS (Earl Bosch Script) project. All project configuration is stored in `project.json`.

## Creating a New Project

When creating a new EBS project, follow these guidelines:

### Project Location
- **All new projects must be created in**: `ScriptInterpreter/projects/`
- **Project documentation must go in**: `ScriptInterpreter/projects/doc/`
- Each project should have its own subdirectory (e.g., `ScriptInterpreter/projects/my-project/`)

### Project Structure
A new project should include:
1. **project.json** - Project configuration file (required)
2. **main.ebs** - Main entry point script
3. **.copilot-instructions.md** - AI assistant instructions for the project
4. **README.md** - Project documentation
5. Additional EBS scripts and resources as needed

### Steps to Create a New Project

1. Create a new directory in `ScriptInterpreter/projects/`:
   ```bash
   mkdir ScriptInterpreter/projects/my-project
   ```

2. Create a `project.json` file with the required configuration:
   ```json
   {
     "name": "My Project",
     "directory": "/path/to/ScriptInterpreter/projects/my-project",
     "description": "EBS Script Project",
     "version": "1.0.0",
     "css": ["console.css"],
     "mainScript": "main.ebs",
     "settings": {
       "autoLoad": true
     }
   }
   ```

3. Create a `main.ebs` script as the entry point

4. Create `.copilot-instructions.md` with AI instructions (copy and modify from existing project)

5. Create `README.md` with project documentation

6. Update `console-projects.json` at the repository root to register the project:
   ```json
   {
     "projects": [
       {
         "name": "My Project",
         "path": "/path/to/ScriptInterpreter/projects/my-project/project.json"
       }
     ]
   }
   ```

### Project Documentation Location
- General project documentation (design docs, specifications, etc.) should be placed in `ScriptInterpreter/projects/doc/`
- Project-specific documentation (README, etc.) should be in the project's own directory

## Project Structure

The project configuration file `project.json` contains all important project properties and directory locations.

### Key Properties in project.json

- **name**: Project name
- **directory**: Project root directory path
- **description**: Project description
- **version**: Project version number
- **mainScript**: Main entry point script file (typically `main.ebs`)
- **css**: Array of CSS files to load for UI styling
- **settings**: Custom project settings

### Directory Organization

The following directories can be configured in `project.json`:

- **resourceDir**: Directory for resource files (images, data files, etc.)
  - Default: `resources`
  - Use this for static assets and resource files

- **testDir**: Directory for test scripts
  - Default: `tests`
  - Place all test EBS scripts here

- **tempDir**: Directory for temporary files
  - Default: `temp`
  - Use this for temporary working files

- **docDir**: Directory for documentation
  - Default: `docs`
  - Place project documentation, markdown files, and guides here

## EBS Script Syntax

For complete EBS (Earl Bosch Script) language syntax and built-in functions, refer to the
EBS interpreter documentation:

- **EBS_LANGUAGE_REFERENCE.md** - Quick reference pointer to syntax documentation
- **docs/EBS_SCRIPT_SYNTAX.md** - Comprehensive syntax reference with all language features

These documentation files are located in the EBS Script Interpreter repository.

The syntax documentation includes:
- Data types and type casting
- Variables, operators, and control flow
- Functions, arrays, and JSON
- Database operations
- Screen/UI windows
- All built-in functions (string, file, HTTP, mail, CSS, array, system, etc.)
- Console commands and best practices

When writing EBS scripts:
- Refer to the syntax documentation for correct language constructs
- EBS is case-insensitive for all identifiers
- Statements typically end with semicolon `;`
- Use `//` for comments

## File Placement Guidelines

When working with this project:

1. **Always read `project.json` first** to understand the project structure and directory layout
2. **EBS Scripts**: Place in the project root or organize in subdirectories
3. **Resources**: Use the directory specified in `resourceDir` property
4. **Tests**: Use the directory specified in `testDir` property
5. **Documentation**: Use the directory specified in `docDir` property
6. **Temporary Files**: Use the directory specified in `tempDir` property
7. **CSS Files**: Reference in the `css` array in project.json

## Working with AI Assistants

When asking AI to create or modify files:
- Specify which directory the file belongs in based on project.json
- AI should check project.json to determine correct file locations
- All paths in project.json are relative to the project root directory
